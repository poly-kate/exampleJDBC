База данных (БД) — именованная совокупность данных,
отражающая состояние объектов и их отношений в рассматриваемой предметной области.
- совокупность данных, хранимых в соответствии со схемой данных, манипулирование которыми выполняют в соответствии с правилами средств моделирования данных.

Не следует путать понятие "база данных" с понятием "система управления базами данных".

Под предметной областью принято понимать некоторую область человеческой деятельности или область реального мира,
подлежащих изучению для организации управления и автоматизации, например, предприятие, вуз и.т.д.

Система управления базами данных (СУБД) — совокупность языковых и программных средств, предназначенных для создания,
наполнения, обновления и удаления баз данных.

1. Реляционная (sql) СУБД представляет собой совокупность таблиц, связанных отношениями.
Популярные sql СУБД: MySQL, PostgreSql, SQLite и тд

2. Не реляционные (nosql) СУБД: данные в таких базах могут хранится разными способами, например
«ключ-значение», документоориентированное хранилище, колоночное хранилище, хранилище на основе графов.
Популярные nosql СУБД: MongoDB, DynamoDB, Redis и тд



JDBC  - платформенно независимый стандарт взаимодействия Java-приложений с различными СУБД, реализованный в виде пакета java.sql.
JDBC Api включает:
DriverManager - класс, использующийся для управления списком драйверов (конкретных реализаций,
которые обеспечивают взаимодействие с той или иной СУБД).
Driver - интерфейс, использующийся для соединения коммуникации с базой данных, управления коммуникации с базой данных.
Connection - интерфейс описывающий соединение с БД.
Statement - интерфейс, включающий методы для построения и выполнения запросов.
ResultSet - набор записей, извлеченных из бд при выполнения запроса.

В своей программы Вы оперируете абстракциями, а конкретные реализации (драйвера) описываете в зависимостях проекта,
именно они и будут обеспечивать реальное взаимодействие с СУБД.
